## Approach

Adds a sixth match model anchored on just the last 4 bytes. The existing match models require a 6-, 8-, ..., 14-byte context to engage, so the many short 4-5 byte repeats common in source code, text and executables (tokens, opcodes, common words) were never picked up by a match model — only modelled statistically. The order-4 model hashes c4, finds the most recent prior occurrence, verifies/extends the match backwards (engaging only when at least 4 bytes actually match), then predicts and follows it exactly like the longer match models, feeding a length-bucketed adaptive StateMap into the mixer.
This is the first change in a while to help the high-entropy binary file (it has many short repeated byte patterns). Round-trip (release + debug) gates pass; every corpus file improves.
