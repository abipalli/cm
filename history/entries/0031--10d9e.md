## Approach

Adds a new model family: indirect context models. For each of the order-1..4 contexts, a table records the recent history of bytes that have *followed* that context (a rolling 4-byte window, packed). The model's context is then hash(context, follow-history) — i.e. 'given this context AND what has recently tended to come next here, predict the next byte'. This captures higher-order regularity that direct fixed-order contexts miss, and is a canonical paq-class component the codec was missing.
- order-1 / order-2 use exact tables (256 / 65536 entries).
- order-3 / order-4 use hashed follow-history tables (2^21 each).
All state is derived purely from already-decoded bytes (updated at each byte boundary before the history rolls), so encode/decode stay identical; each indirect context carries its own counter + bit-history StateMap into the mixer. Release and debug round-trip gates pass; text_reymont, struct_xml, repetitive_nci and source_samba all improve markedly. Release test (4 threads) peaks ~9.1 GB on the 16 GB runner.
