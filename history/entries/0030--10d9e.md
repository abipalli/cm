## Approach

Adds a new model family: 2D / 'byte-above' modelling, which predicts from the byte at the same column in previous lines/records. This structure is invisible to the existing 1D context, sparse, and match models, yet it is highly predictive for aligned source code, text, tables and XML.
Newline-based 2D (tracks line starts deterministically):
- byte directly above (same column, previous line) + current column
- byte above + byte to the left (2D neighbourhood)
- byte above + byte above-left (diagonal)
- byte two lines up + byte above (vertical bigram)
- a 12th layer-1 mixer specialist selected by the byte-above value
Record model (for newline-free periodic data such as executables/tables): a deterministic Boyer-Moore majority vote over per-byte recurrence distances estimates the dominant period R, then two contexts read the byte one period back (+ left byte, + the preceding period-back byte).
All state is derived purely from already-decoded bytes, so encode/decode stay identical; each context carries its own counter + bit-history StateMap. Release and debug round-trip gates pass; repetitive_nci, source_samba, struct_xml and both text files all improve (binary_mozilla has no line/period structure so is ~flat — inherent). Release test (cargo test --release, 4 threads, matching the 16 GB/4-vCPU public runner) peaks ~8.7 GB.
