## Approach

Output-neutral WORK reduction in the mixer, which profiling shows is ~74% of total WORK. The 27 layer-1 specialist mixers each dot the same 221-element input vector (mix_in64) under their own context-selected weight row. Running them as 27 separate loops re-read the whole input vector 27 times per bit. This fuses them: first set each mixer's selected ctx, pre-slice its weight row (rows[k] = &l1[k].w[base..base+n]), then run a single pass over the inputs — loading each mix_in64[i] once — accumulating into all 27 dot products at once. Each dot is still sum_i w_k[i]*x[i] in the same order, so every layer-1 logit, the cached pr, and the compressed output are byte-for-byte identical: SCORE 572577 (record), all six per-file sizes unchanged; WORK 9.88G -> 9.13G (-7.62%) on scripts/measure-complexity.sh; round-trip suite passes. The layer-2 path is unchanged.
Builds on the prior PR that pre-widened the inputs to i64 (the fused loop reuses that widened vector).

## Iteration notes

The layer-1 *update* path (same structure) is a candidate for the same fusion next. SCORE 572577 unchanged; the record's WORK tiebreak drops substantially.
