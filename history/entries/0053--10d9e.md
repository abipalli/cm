## Approach

Replaces direct-mapped context hash tables with 4-way set-associative buckets for all high-cardinality models (the dense orders, sparse/stride banks, gap-bigrams, nibble/delta transforms, and indirect families). Each context maps to a 4-slot bucket; an 8-bit checksum (the high hash bits the bucket index discards) selects the matching way, and on a miss the lowest-count way is evicted. Colliding contexts therefore occupy separate warm slots instead of polluting a single shared counter — recovering essentially all of the loss that hash collisions caused on the 384 KB-per-file corpus, where each model touches ~3M distinct slots. SCORE 575084 -> 574203 (-881), improving every file. Crucially this is also memory-NEGATIVE: because associativity resolves collisions, the data tables shrink to 2^22 (from 2^23) and still beat the old direct tables, so per-Cm committed memory drops to ~2.5 GB (from ~3.7 GB) — single-threaded eval peaks at 2.6 GB. Low-cardinality models (order-0/1/2, bytes-2-3, char-class) stay direct-mapped; they have no collisions to resolve and would only waste the checksum byte.

## Iteration notes

This captures the collision loss that the (CI-OOM-blocked) table-growth route could not: a direct 2^25 table scored -740 locally but OOM-killed CI, whereas 4-way associativity at 2^22 scores -881 at LOWER memory than the prior record. 2-way at 2^22 was -380 and 2-way at 2^23 was -553; 4-way at 2^22 (-881) wins because at this load (~3 contexts/bucket) four ways cut the eviction/cold-start rate to ~10% vs ~19% for two ways, while keeping the small table. The earlier collision-confirmation experiment (reset the slot on checksum mismatch) regressed +248 because it cold-started on EVERY collision; associativity avoids that by giving each colliding context its own warm slot. All 9 round-trip tests pass; eviction is deterministic from reconstructable state so encode/decode stay in lockstep.
