## Approach

Raises the prediction/coding precision from 12-bit to 16-bit through the final SSE/APM chain and the arithmetic coder. Previously the combined mixer logit was squashed to a 12-bit probability (0..4095) and every APM stage re-quantized to that 1/4096 grid, capping how confident a prediction could be coded (a near-certain bit still cost ~-log2(4094/4096) ~= 7e-4 bits). Now the combined logit is squashed via a new 16-bit logistic table (squash16), the four APM stages interpolate and blend at 16-bit (their calibration tables were already ~16-bit internally — only the output was being truncated), and the coder uses range*p>>16 with p in [1,65534]. A confident bit now codes at ~-log2(65534/65536) ~= 4e-5 bits. SCORE 574135 -> 573541 (-594), improving every file; the gain is broad because near-deterministic bits are everywhere in a strong model. Memory-free (two small logistic tables, ~0.5 MB) and lossless (all 9 round-trip tests pass; the higher-precision p is still deterministic on both sides).

## Iteration notes

Built incrementally and verified at each step: (1) only the final APM stage + coder at 16-bit = -138; (2) the WHOLE chain at 16-bit (squash16 entry + all four APMs via a 16-bit apply + 16-bit blends) = -594. Widening the mixer logit clamp beyond +/-2047 was considered but is now near-worthless: at 16-bit the confidence cap is already ~0.99997, so there is no meaningful headroom left there. The per-context counters/statemaps and the mixer dot stay 12-bit (their stretch inputs are unchanged); only the post-mixer probability pipeline went 16-bit. Removed the now-unused 12-bit apply().
