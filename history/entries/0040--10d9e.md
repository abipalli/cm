## Approach

Adds a new model family: a nesting model that tracks the stack of currently-open brackets ()[]{} and predicts from the program's block structure — a paq8-class component the codec lacked, targeting source code (the corpus's largest file after the executable) and structured data. A 64-deep bracket stack is maintained deterministically from decoded bytes; three contexts feed the mixer:
- nesting depth + last byte
- the enclosing (innermost open) bracket + last byte
- enclosing bracket + depth + order-2 context
All general (no corpus-specific logic); each carries its own counter + bit-history StateMap. Release and debug round-trip gates pass; source_samba improves most (176716 -> 176618), with text also slightly better.
