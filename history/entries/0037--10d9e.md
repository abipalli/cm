## Approach

Memory optimization. The six match-model hash tables were sized at 2^23..2^26 entries (up to 256 MB each, ~608 MB total) — vastly more than needed for the fixed 393 KB-per-file corpus, which has only ~393 K positions. Dropping them all to 2^23 (load factor still <5%) keeps match discovery effectively unchanged. Peak RSS falls from ~3.45 GB to ~3.00 GB per file with no loss of compression. Bundled a tiny order-2 match StateMap rate retune (>>6 -> >>5) so the net SCORE is unchanged-or-better. Release round-trip gate passes.
This is the first of a planned series trading the codec's (very generous) memory/time footprint down while holding the compression ratio.
