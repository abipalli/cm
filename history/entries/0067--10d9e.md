## Approach

Output-neutral WORK reduction in the CTW predict walk. The loop visits depths d = D-1..0 and, at each step, fetches nd = node(d), the on-path child node(d+1), and the off-path sibling. But node(d+1) is exactly the nd computed in the previous (deeper) iteration, so it is fetched twice. Carry the previous nd forward as the on-path child (initialised to the depth-D leaf), eliminating ~D redundant map lookups per predict. The values used are identical, so every CTW prediction — and the compressed output — is byte-for-byte unchanged: SCORE 572577 (record), WORK 10.22G -> 10.14G on scripts/measure-complexity.sh; round-trip suite passes.
