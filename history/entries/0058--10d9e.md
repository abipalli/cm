## Approach

Pure efficiency change: SCORE is byte-for-byte identical (572769); the codec is ~20-25% faster. The per-context tables were stored as four parallel arrays (cp probability, cn count, st bit-history state, ck checksum), so every model's slot access touched up to four separate multi-GB regions = four cache-line misses per bit, and the model bank is the memory-bound hot path. They are now a single array of a packed 5-byte Slot { cp:i16, cn:u8, st:u8, ck:u8 } per model, so an 8-way associative bucket lives in ~one cache line and a slot read/update is a single miss. Same total memory (5 bytes/slot either way), same arithmetic, same output — verified: every corpus file is byte-identical and all 9 round-trip tests pass. Local single-file compress dropped from 17.7s to 13.8s user time; the win should be larger on the more memory-latency-bound CI runner.

First of a planned speed pass (the model has grown very heavy: ~106 context models + 27 mixer specialists, eval ~230s/6 files).
