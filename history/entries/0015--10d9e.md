## Approach

Doubles each context model's hash table from 2^22 to 2^23 slots. With the context bank now at 55 models and inputs processed bit-by-bit (index = ctxhash*769 + partial-byte), the high-order and strided-sparse models were running their 4M-slot tables at a high load factor on 384 KB inputs, so hash collisions were corrupting predictions. Halving the load factor materially reduces those collisions.
Single constant, fully general. Peak RSS ~2.4 GB (well within the CI runner's budget). Release and debug round-trip gates pass; every corpus file improves.
