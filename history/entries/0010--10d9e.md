## Approach

Replaces the single context-selected logistic mixer with a two-layer mixing network — the largest remaining lever called out in AUTORESEARCH and not previously attempted.
Layer 1 is five specialist mixers, each running the full 57-input vector but selected by a different context:
- **m0** — the proven last-byte + match-activity context (16384 rows).
- **m1** — the within-byte partial-byte context `c0` (order-0 bit position, 256 rows).
- **m2** — the second-to-last byte (an order-2-distance specialist, 256 rows).
- **m3** — the full last two bytes (order-2, 8192 rows).
- **m4** — a hash of the last three bytes (order-3, 8192 rows).
Layer 2 is a small combiner that mixes the five layer-1 logits, keyed on the last byte (256 rows). Each mixer trains online on its own prediction error (standard PAQ-style per-mixer training), so the layer-1 specialists each see the global signal while the layer-2 combiner learns how much to trust each one in a given context.
All contexts are general (order-0..3 byte history + match activity) — no corpus-specific logic. Every corpus file improves; the two-layer structure adds capacity without the undertraining that killed earlier single-mixer-context expansions, because each specialist keeps a focused selection context and the combiner has only five inputs. Release and debug round-trip gates pass.
