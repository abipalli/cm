## Approach

Memory optimization with provably identical output. The order-0 context model has ctxhash[0] == 0, so its table index is just the partial-byte value c0 (always <=255) — yet it was allocated a full 2^23-slot table like every other model (~32 MB). Introduces a per-model table-size mask and shrinks order-0 to 512 slots; since c0 & 511 == c0, every index is unchanged, so the predict/update sequence and SCORE are byte-for-byte identical. All other models keep the full table. Saves ~32 MB with zero compression cost. Round-trip gate passes; SCORE exactly 578672.
(Per the maintainer's 'keep ratio exact' directive: this is strictly SCORE-neutral. The larger context tables are load-bearing — shrinking them trades ratio — and SCORE-neutral speedups are blocked by the baseline-x86 build flags, so this is near the memory floor at fixed ratio.)
