## Approach

Widens the set-associative context tables (added in the previous record) from 4 ways to 8 ways per bucket, at the same 2^22-slot table size (so memory is unchanged). At this corpus's bucket load, four ways still evict/cold-start on ~10% of accesses; eight ways cut that further, keeping more colliding contexts in warm slots. The probe is a short loop that breaks on the first checksum match (so average work barely changes) and evicts the lowest-count way on a miss. SCORE 574203 -> 574135 (-68), improving across files. Memory-neutral: ~2.5 GB/Cm committed, single-eval RSS ~2.6 GB.

## Iteration notes

Growing the data table to 2^23 (4-way) gained only -93 but pushed eval RSS to 4.5 GB, not worth the OOM risk, so kept 2^22.
