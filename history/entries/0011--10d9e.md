## Approach

Adds a third APM/SSE calibration stage after the existing two, keyed on a *dense* context: the within-byte partial-byte value `c0` (bit position + high bits already seen) combined with two match-activity flags (short order-6 match active, long order-10 match active), for 1024 contexts.
Entry 0007 had rejected a third APM, but that attempt used a *sparse* order-2 byte context (65536 entries) that undertrained on 384 KB inputs. This stage is the opposite: `c0` takes only ~255 dense values that every byte exercises hundreds of thousands of times, so the table calibrates fast and corrects the systematic per-bit-position bias the mixer leaves behind. Splitting by match activity lets the calibration differ between in-match and out-of-match regions. Blended like the others (`p = (p + 3*a) >> 2`).
General calibration only — no corpus-specific logic. Every corpus file improves (e.g. source_samba 180098 -> 179612, text_dickens 93861 -> 93489, binary_mozilla 232843 -> 232584). Release and debug round-trip gates pass.
