## Approach

Output-neutral WORK reduction in the CTW model (the dominant WORK contributor). For each bit the codec calls predict() then update(); both walk the same depth-0..D path over the same history, and nothing writes the node store between them. So update() was re-fetching from the HashMap every node predict() had just read. predict() now stashes the path node at each depth (spath[0..=D]) and each sibling's lw (ssib[0..D]); update() reads those stashes instead of calling get(), removing ~65 map lookups per bit. update() still recomputes the KT/Pe/Pw values and inserts the updated nodes, so the stored state is byte-for-byte the same on both encoder and decoder (they run the identical predict-then-update sequence). SCORE stays 572577 (record); WORK 10.14G -> 9.98G (-1.59%) on scripts/measure-complexity.sh; all round-trip tests pass.
This is sound because predict and update are always paired (predict first) and the history is unchanged between them, so the stashed values equal what update's get() would return.
