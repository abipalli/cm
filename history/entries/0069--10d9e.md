## Approach

Output-neutral WORK reduction in the mixer, which profiling shows is ~74% of total WORK (the dominant cost). The layer-1/2 dot products computed 'wi as i64 * xi as i64', sign-extending both operands inside the inner loop. But the input vector is the same for every mixer (all 27 layer-1 mixers dot mix_in; all 10 layer-2 mixers dot the 27 layer-1 logits), so each input was sign-extended ~27x per bit. Pre-widen the inputs once per bit into mix_in64 / l2_in64 (a single NINPUT- and NL1-length pass), and have mix() take &[i64] so the hot loop does 'wi as i64 * xi' with xi already widened. The weights stay i32 (they wrap at i32 by design, so they must), so one extend per MAC remains; the per-input extend is gone. Values and accumulation order are unchanged, so output is byte-identical: SCORE 572577 (record), all six per-file sizes unchanged; WORK 9.98G -> 9.88G (-1.03%) on scripts/measure-complexity.sh; round-trip suite passes.

## Iteration notes

Profiled the WORK breakdown by stubbing components: mixer ~7.4G (74%), CTW ~1.0G (10%, already optimized -20% in prior PRs), rest ~1.6G. The remaining mixer cost (i32 weight extend + i64 MAC) is irreducible without changing the weight wrapping (which would change output). SCORE 572577 unchanged.
