## Approach

1. **Second match model at order-8.** Alongside the existing order-6 match model,
   a long-match specialist with its own hash table (`mmtab2`), bit-history state
   map (`mm_sm2`), and a dedicated mixer input. It only engages on genuinely long
   repeats (matched run >= 8 bytes), so it complements rather than duplicates the
   order-6 model. The mixer context now selects weights on both match flags
   (bits 8 and 9), letting the mixer learn how much to trust each match model.
2. **Order-7 context model.** Added a tenth context model hashing the last 7
   bytes (`ctxhash[9]`), extending the bank from orders 0-6 to orders 0-7 (plus
   word and sparse). Helps higher-order text and source structure.
Both changes are general compression: no corpus-specific logic, no side channels,
fully deterministic (encode/decode run the identical predict/update sequence).
All `wrapping_*` ops are used on hashes to stay overflow-safe in debug builds.
