## Approach

Adds three general-purpose shape/layout context models to the existing context mixer and slightly retunes retained match-table sizes:
- Adds a word-shape context combining the current rolling word hash with the previous byte, with a compact sparse fallback for non-word regions.
- Adds a second text-shape context using case-folded local bytes for word regions and a compact recent-byte blend otherwise.
- Tracks a deterministic line column counter and adds two layout contexts keyed by column, recent bytes, byte class, and word hash.
- Keeps the useful table-size retunes found during iteration: order-8 match table at `2^26`, order-12 match table at `2^24`.
These are general compression signals for text, source, XML, binary byte-shape, and repetitive records. They use only deterministic encoder/decoder state and do not inspect corpus identity or external state.
